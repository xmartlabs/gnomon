# AQ v2 — 4-pillar Agentic Quotient + gstack/AQ separation

_Design spec · 2026-06-10 · extends `2026-06-09-paxel-xl-aq-design.md`_

## Objetivo

Expandir AQ de "1 score, 4 ejes" a **4 pilares** y dejar explícito en el reporte que son dos preguntas distintas:

- **gstack scorecard + archetype** = "qué tan buen **builder** sos" (craft de código; sin cambios de cómputo).
- **AQ** = "qué tan buen **operador de agentes** sos" (multidimensional).

Decisión de framing (Opción 1, "lentes complementarios"): NO se toca el cómputo de los 3 scores gstack ni del archetype. Se reposiciona en el layout/copy y se acepta solapamiento *enmarcado* (misma evidencia, lente distinta). No se borran métricas por solापar.

## Estructura AQ v2

`compute_aq(stats)` devuelve:
```
{
  aq_0_100,                 # overall ponderado
  tier,                     # mismos cortes: <40 Operator, <60 Power User, <80 Orchestrator, >=80 Systems Builder
  pillars: [
    {name, weight, score,   # score 0-100 = 100 * weighted-avg de sub01 de sus axes
     axes: [{name, weight, score, signals}]},
    ...
  ],
  mcp_vs_cli, tool_diversity,  # descriptivos, sin cambios
}
```
- **Modelo de pesos:** dentro de cada pilar, cada axis tiene un `weight` en PUNTOS que suman 100 por pilar. `axis.score = round(weight * sub01, 1)` (0..weight). `pillar.score = round(sum(axis.score), 1)` → 0..100. Así "score/weight" se lee parejo en todos lados.
- **Overall:** `aq_0_100 = round(.30*Breadth.score + .35*Craft.score + .20*Efficiency.score + .15*Savvy.score)`.
- Pesos de pilar: **Breadth 30 · Craft 35 · Efficiency 20 · Savvy 15** (Craft pesa más: calidad > cantidad).

### Pilar 1 — Breadth (30)
Los 4 ejes actuales, sin cambios de fórmula ni de pesos (ya suman 100): orchestration 33, skill fluency 22, tool command 28, discipline 17.

### Pilar 2 — Craft (35)
Pesos (suman 100): verification 40, grounding 30, compounding 30.
- **verification** = .5·sat(shell_test_runs, 150) + .5·sat(review_skill_uses, 100)
  donde `review_skill_uses = _skill_uses_any(stats, ["code-review","cerberus","verify","requesting-code-review","review"])` sobre `skills_all`.
- **grounding** = sat(planning_ratio_explore_to_doing, 1.0)
- **compounding** = .6·sat(compounding_writes, 30) + .4·(1.0 si `_skill_uses_any(["retro","writing-plans","brainstorm"])` else 0.6)

### Pilar 3 — Efficiency (20)
Pesos (suman 100): steering_leverage 50, recovery 50.
- **steering_leverage** = curva sweet-spot sobre `actions_per_prompt` (app), banda ideal [5, 20]:
  - `5 <= app <= 20` → 1.0
  - `app < 5` → app/5
  - `app > 20` → max(0.0, 1 - (app-20)/40)   (app≥60 → 0)
- **recovery** = .85·error_recovery_ratio + .15·(1 - sat(api_errors_retries, 50))

### Pilar 4 — Savvy (15)
Pesos (suman 100): model_tier 50, token_economy 50.
- **model_tier** = .5·sat(distinct_model_families, 3) + .5·sat(nonopus_share, 0.30)
  - families: colapsar `stats["stack"]["models"]` por regex → "opus"/"sonnet"/"haiku"/otro (e.g. `claude-opus-4-7` y `-4-8` → "opus").
  - nonopus_share = turns sin "opus" en el nombre / total turns.
- **token_economy** = .5·sat(toolsearch_calls, 300) + .5·sat(cli_share, 0.70)
  - cli_share = cli_calls / (cli_calls + mcp_calls) si denom>0 else 0.

## Data collection nueva

1. **`compounding_writes`** (colector nuevo): en las ramas Edit / Write / MultiEdit / NotebookEdit del loop, cuando el `file_path`/`notebook_path` matchea (case-insensitive) `r"CLAUDE\.md|AGENTS\.md|GEMINI\.md|/memory/|/docs/adr|\.cursorrules"`, incrementar un `compounding_counter`. Exponer `stats["stack"]["compounding_writes"]`.
2. **`skills_all`**: `stats["stack"]["skills_all"] = skill_counter.most_common()` (lista completa, no solo top-15) para que la detección de skills (v1 meta + v2 verification/compounding) no dependa del truncado. Actualizar `_skill_uses`/`_skill_uses_any` (o las closures locales en compute_aq) para leer `skills_all` con fallback a `top_skills`.
3. Resto: ya existe (`shell_test_runs`, `planning_ratio_explore_to_doing`, `error_recovery_ratio`, `api_errors_retries`, `actions_per_prompt`, `toolsearch_calls`, `cli_calls`, `mcp_calls`, `models`).

- Mantener split MCP-vs-CLI + tool diversity.

`stats["agentic"]` ahora con la estructura de pilares.
